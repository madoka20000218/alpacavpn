# Интерфейс Telegram-бота Alpaca VPN

Подробное описание интерфейса пользователя и администратора. **Этот документ — единственный source of truth по UX.** Любые изменения текстов, кнопок, сценариев сначала правятся здесь, и только затем переносятся в `locales/ru.yaml` и код.

Все решения, принятые по умолчанию в ходе переписывания. Менять — только через ADR (`docs/adr/*.md`), как описано в `STACK.md` §5.1.

## 1. Общие принципы

### 1.1. Визуальные правила
- Все меню используют **inline-кнопки**. `ReplyKeyboard` не используем.
- Каждое меню (кроме меню оплаты и уведомлений из §12) имеет **баннер-изображение** — PNG 1280×720 до 512 КБ (`STACK.md` §5.1).
- Все цены — в рублях (`₽`). В меню оплаты дополнительно показывается цена в Telegram Stars (`XTR`) — конвертация по `tariffs.price_xtr`.
- Тексты parse_mode = `HTML` везде. Теги: `<b>`, `<i>`, `<code>`, `<a href>`. Никогда не кладём сырой `first_name`/`username` в HTML — только через `html.escape()`.

### 1.2. Навигация
- Кнопка `Назад` возвращает в предыдущее меню по стеку, сохранённому в `state.data["nav"]` (см. `ARCHITECTURE.md` §11.4).
- Команда `/start` **всегда** очищает FSM и возвращает в главное меню, даже если пользователь был в середине флоу (`await state.clear()`).
- Команда `/help` показывает §11 (Помощь), флоу не сбрасывается.
- В любой момент `/admin` (только админ) входит в §14.1 с очисткой FSM.

### 1.3. Отправка секретных данных
- Все **VLESS-ключи** и **строки прокси** отправляются отдельным сообщением в `<code>…</code>` (тап → копирование).
- **Реферальные ссылки** также в `<code>…</code>`.
- При отправке прокси перед строкой отдельное предупреждение: «Никому не пересылайте эти данные.»
- Секретные данные никогда не попадают в логи и Sentry (§16 `ARCHITECTURE.md`).

### 1.4. Обработка ошибок
- При любой ошибке пользователю отправляется текст из §13 с кнопкой `Назад`.
- Забанённый пользователь на любое действие получает «Вы заблокированы.», callback-нажатия также перехватываются.
- Клавиатура без кнопки `Назад` встречается только в §2 (главное меню) и в меню оплаты (кнопка завершает выбор или отменяет инвойс).

## 2. Главное меню

- **Команда запуска:** `/start` (также реагирует на deeplink `/start <referral_code>` — см. §10)
- **Баннер:** `glavmenu`
- **FSM:** `/start` очищает состояние (`state.clear()`).

**Текст:**

```
Добро пожаловать в Alpaca VPN!

Статус:    Безопасно
Протокол:  VLESS

Используйте кнопки ниже для навигации.
```

> «Статус» в MVP — статический текст «Безопасно» (ADR в STACK §5.1 п.7). Задел: поле `health_status` в renderer может вычисляться в будущем из пингов 3x-ui.

**Inline-кнопки:**

| Ряд | Кнопки | Callback data |
|---|---|---|
| 1 | `Купить подписку` · `Купить прокси` | `menu:buy_vpn` · `menu:buy_proxy` |
| 2 | `Мои подписки` · `Инструкция` | `menu:subs` · `menu:inst` |
| 3 | `Партнёрам` · `Помощь` | `menu:partner` · `menu:help` |
| 4 | `Продлить подписку` · `Подарить другу` | `menu:extend` · `menu:gift` |

> **Кнопки 4 ряда** (Продлить/Подарить) показываются **всегда** (даже если нет активной подписки): «Продлить» покажет ошибку §13 («У вас нет активной подписки»), «Подарить» — работает всегда.

### 2.1. Триал для новых пользователей

Новый пользователь при `/start` (если `users.trial_used = false`, т.е. триал ещё не активирован) получает **вторым сообщением** после главного меню:

```
В подарок за регистрацию — 3 дня VPN бесплатно!
```

**Inline-кнопки:** `Активировать 3 дня` · `Позже`

**Поведение «Активировать 3 дня»:**
1. Редис-лок `trial:lock:{user_id}` TTL 30 с от двойного клика;
2. Проверка `users.trial_used` — если `true`, ошибка §13 («Триал уже активирован»);
3. `TrialService.activate(user)`: создаёт `subscription(source='trial', duration=3d)`, запускает сагу `provision_vpn` (см. `ARCHITECTURE.md` §8.5);
4. `users.trial_used = true`;
5. Outbox: `trial_activated` → пользователь получает уведомление + ключ (§12).

**Поведение «Позже»:**
- Сообщение удаляется (`delete_message`), FSM остаётся чистым.
- Предложение выдаётся при каждом последующем `/start` до тех пор, пока триал не активирован.

## 6. Мои подписки

- **Баннер:** `sub`
- **Показывает все подписки пользователя**, сгруппированные по статусу (`active` → `pending` → `failed`). «Истёкшие» (`expired`, `revoked`) не показываются (это архив).

### 6.1. Варианты текста

**АВАРИЯ 0 — нет никаких подписок:**

```
У вас нет активных ключей.

Купите подписку в главном меню или активируйте подарок от друга.
```

**ВАРИАНТ 1 — есть только `active` ключи:**

```
Активные ключи доступа:

1. {вид подписки}
   Действует до:   {дата окончания}
   Осталось дней:  {количество}
```

Следом — отдельное сообщение с самим ключом в формате `<code>{ключ VPN}</code>` (тап → копирование).

**ВАРИАНТ 2 — есть `pending` ключ (оплата прошла, 3x-ui недоступен):**

```
Активные ключи доступа:

1. {вид подписки}
   Статус:         Готовим ключ… ⭐️
   Приблизительно:  {до 15 минут}

Мы выдадим ключ автоматически — отдельным сообщением, как только панель
станет доступна. Повторная оплата не требуется.
```

**ВАРИАНТ 3 — есть `failed` ключ (провижининг не удался 24+ часа):**

```
Активные ключи доступа:

1. {вид подписки}
   Статус:  Не удалось выдать ключ ⚠️

Напишите в поддержку, мы разберёмся вручную.
```

### 6.2. Правила отображения

- Варианты могут смешиваться: у пользователя может быть 1 `active` VPN + 1 `pending` прокси.
- Ключ (`<code>vless://…</code>`) отправляется только для `active` подписок. Для `pending`/`failed` ключа ещё нет.
- Для прокси: вместо ключа — строка `<code>host:port:login:password</code>` + предупреждение (§1.3).
- Если подписка истекает в ≤ 3 дня — рядом с подпиской появляется «⚙️ Скоро истекает».

### 6.3. Inline-кнопки

| Ряд | Кнопки | Примечание |
|---|---|---|
| 1 | `Инструкция` | переход в §9 |
| 2 | `Продлить` · `Подарить другу` | «Продлить» активна только при наличии `active`/`pending` VPN-подписки |
| 3 | `Написать в поддержку` | показывается только при наличии `failed` ключа |
| 4 | `Назад` | в главное меню |

## 10. Партнёрам

- **Баннер:** `partner`

**Текст:**

```
Партнёрская программа

Приглашайте друзей и получайте бонусные дни подписки!

Условие: за каждых 2 приглашённых друзей, которые КУПЯТ подписку,

Ваша статистика:
├ Приглашено всего:             {количество}
├ Купили подписку:              {количество}
├ Бонусных дней получено:       {количество}

```

Следом отдельным сообщением — ссылка в формате `<code>{реферальная ссылка}</code>` (тап → копирование).

**Inline-кнопки:**

| Ряд | Кнопки | Реализация |
|---|---|---|
| 1 | `Поделиться` | `switch_inline_query` с пред-заполненным текстом и ссылкой |
| 2 | `Назад` | — |

**Уведомления реферреру:**
- При переходе и регистрации друга: `Ваш друг @{username} присоединился по вашей ссылке.`
- При **первой платной** активации подписки другом: `Ваш друг @{username} купил подписку. Ещё {1 или 2} — и вы получите бонусный день.`
- При начислении бонуса: см. §12 (`referral_bonus`).

### 10.1. Правила начисления бонуса (антифрод)

| # | Правило | Обоснование |
|---|---|---|
| 1 | Бонус начисляется **только за первую платную** активацию друга | Триал бесплатный → фермы ботов дали бы неумеренные бонусы |
| 2 | Реферрер `≠` рефери | Очевидно |
| 3 | `referee.referrer_id` заполняется **только при первом входе** (если `users.id` уже существует, deeplink ref игнорируется) | Защита от смены реферрера пост-фактум |
| 4 | Если реферрер забанен на момент активации рефери — бонус не начисляется | Банутый не должен получать выгоду |
| 5 | **Max 30 бонусных дней/сутки** на одного реферрера | Защита от вирусных промокодов; счётчик в Redis `referral:quota:{user_id}:{YYYY-MM-DD}` |
| 6 | Счёт по `referrals.first_paid_activation_at IS NOT NULL AND counted_for_bonus=false` | Однозначная семантика «платная активация» |
| 7 | При достижении 2 неучтённых → `bonus_days = count // 2`, помечаем `counted_for_bonus=true`, продлеваем/создаём бонусную подписку | См. §6.5 `ARCHITECTURE.md` |

### 10.2. Поле «До следующего бонуса»

Вычисляется как `2 - (uncounted_paid_activations % 2)`, где `uncounted_paid_activations` — число рефери с `first_paid_activation_at IS NOT NULL AND counted_for_bonus=false`.

### 12.5. Правила доставки

- **Дедупликация.** `UNIQUE(dedup_key)` на `outbox` — повторная запись с тем же ключом игнорируется.
- **Приоритеты.** Flusher выбирает записи `ORDER BY priority ASC, id ASC`. Транзакционные (10) вытесняют рассылки (200) при насыщенной очереди.
- **Rate-limit.** Bucket `tx` — 30 msg/s (приоритет 10/50/100), bucket `broadcast` — 25 msg/s (приоритет 200). Общий потолок — 30 msg/s на бота.
- **Безопасность.** Секреты (`{vless_url}`, `host:port:login:password`, `referral_link`) **всегда** идут отдельным сообщением в `<code>…</code>` (§1.3).
- **Ошибки Telegram.** `TelegramRetryAfter` → `available_at = now() + retry_after`, `attempts += 1`. `TelegramForbiddenError` (пользователь заблокировал бота) → `status='failed'`, в поддержку не пишем.

## 13. Обработка ошибок

Все доменные исключения наследуются от `DomainError` (`ARCHITECTURE.md` §4.3). Центральный `errors_router` aiogram ловит `DomainError`, преобразует в текст по таблице и отвечает пользователю. Прочие исключения → Sentry + `err.generic`.

| Ситуация | i18n_key | Сообщение | Domain exception | Кнопки |
|---|---|---|---|---|
| Пользователь не найден | `err.user_not_found` | `Пользователь не найден. Проверьте @username или ID.` | `UserNotFoundError` | `Назад` |
| Недостаточно средств | `err.insufficient_funds` | `Недостаточно средств для оплаты. Попробуйте другой способ.` | `PaymentDeclinedError` | `Назад` |
| Нет подписки для продления | `err.no_subscription` | `У вас нет активной подписки. Купите новую в главном меню.` | `NoActiveSubscriptionError` | `Купить подписку` · `Назад` |
| Нет прав администратора | `err.no_admin_access` | `У вас нет доступа к этой функции.` | `PermissionDeniedError` | `Главное меню` |
| Счёт истёк (15 мин) | `err.invoice_expired` | `Счёт истёк. Создайте новый.` | `InvoiceExpiredError` | `Назад` |
| Получатель подарка не запускал бота | `err.gift_target_not_found` | `Получатель ещё не запускал Alpaca VPN. Попросите его нажать /start и повторите.` | `GiftTargetNotFoundError` | `Назад` |

| Триал уже активирован | `err.trial_already_used` | `Триал уже активирован. Купите подписку в главном меню.` | `TrialAlreadyUsedError` | `Купить подписку` · `Назад` |
| Самоприглашение (`/start?ref=<собственный_код>`) | `err.ref_self` | `Нельзя пригласить самого себя.` | `ReferralSelfError` | `Главное меню` |

| Слишком много запросов (anti-flood) | `err.too_many_requests` | `Слишком много запросов. Подождите немного.` | — (ThrottleMiddleware) | — |
| Сумма в вебхуке не совпала с инвойсом | `err.payment_mismatch` | `Платёж отклонён: сумма не совпала. Обратитесь в поддержку.` | `PaymentMismatchError` | `Написать в поддержку` |
| Техническая ошибка (generic) | `err.generic` | `Произошла ошибка. Попробуйте позже или обратитесь в поддержку.` | остальные `DomainError` | `Назад` |

### 13.1. Правила

- **Кнопка `Назад`** прилагается к каждому сообщению, кроме `too_many_requests` (там только текст, чтобы не раздражать флудера).
- **`TelegramRetryAfter` / `TelegramNetworkError`** пользователю **не показываются** — только лог + ретрай в outbox.
- **`DomainError`** → структурированный лог `structlog` с `request_id`, `user_id`, `error_code` и `handler`.
- **`ProvisioningUnavailableError`, `err.generic`** → дополнительно в Sentry (критичность — WARN/ERROR).
- **Секреты** (`{vless_url}`, `proxy.password`) никогда не попадают в тексты ошибок.

#### 14.7.1. Изменить цены

Выбор тарифа → запрос новой цены → подтверждение → применение (с записью в лог).

#### 14.7.2. Изменить баннеры

Выбор меню (`главное`, `тарифы`, `прокси`, `инструкция`, `подарок`, `партнёры`, `помощь`, `продление`) → запрос нового изображения → применение.

> **Требования к баннеру:** *уточнить формат и разрешение*.

#### 14.7.3. Управление админами

- Список текущих админов.
- Inline-кнопки: `Добавить админа` · `Удалить админа`.
- Добавление: запрос `@username` или ID → проверка → добавление.

#### 14.7.4. Перезапуск бота

- Подтверждение: `Перезапустить бота? Все активные сессии будут сброшены.`
- Inline-кнопки: `Да, перезапустить` · `Отмена`.

#### 14.7.5. Логи

- Последние 100 строк лога с фильтрами.
- Inline-кнопки: `Ошибки` · `Платежи` · `Входы` · `Назад`.

## 15. Потоки пользователя (User Flows)

**Покупка подписки:**
`/start` → Главное меню → `Купить подписку` → Выбор тарифа → Меню оплаты → Оплата → Успех → Ключ

**Покупка прокси:**
`/start` → Главное меню → `Купить прокси` → Тип и срок → Меню оплаты → Оплата → Выдача `host:port:login:password`

**Триал:**
`/start` (первый раз) → `Активировать 3 дня` → Ключ → Уведомление

**Продление:**
`/start` → Главное меню → `Продлить подписку` → Выбор периода → Меню оплаты → Оплата → Успех → Уведомление

**Подарок:**
`/start` → Главное меню → `Подарить другу` → Выбор тарифа → Ввод получателя → Подтверждение → Меню оплаты → Оплата → Уведомление получателю

**Партнёрская программа:**
`/start` → Главное меню → `Партнёрам` → `Поделиться` → Друг регистрируется → Друг активирует подписку → Бонус

**Админ: выдача подписки:**
`/admin` → `Управление подписками` → `Выдать подписку` → Ввод получателя → Выбор тарифа → Подтверждение → Уведомление пользователю
